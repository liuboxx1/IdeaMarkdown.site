IdeaMarkdown 文档

IdeaMarkdown 是一款桌面 Markdown 编辑器：编辑当前区块时显示源码，其余内容以排版后的形式展示，兼顾写作、笔记与轻量技术文档编辑。

本手册涵盖安装、编辑器功能、主题与导出，以及通过 MCP 让 AI 助手直接操作当前文档的完整用法。

安装

前往 下载页 获取 Windows 安装程序（.exe）。当前版本 v0.1.4。安装时你可以自定义安装位置；如果不改，NSIS 安装程序的默认路径为 C:\Users\<用户名>\AppData\Local\IdeaMarkdown\。

安装包会同时附带 AI 控制所需的 ideamarkdown-mcp.exe，与主程序位于同一目录，无需单独下载。

安装完成后，双击任意 .md / .markdown 文件即可用 IdeaMarkdown 打开（已注册文件关联）。

编辑与区块

文档由一系列区块组成：标题、段落、列表、任务清单、引用、表格、代码块、公式与 Mermaid 图表。当光标进入某个区块时显示其 Markdown 源码，离开后立即渲染为排版态。

  • 拖动重排：左键拖动区块首部的把手可整体重排区块顺序。
  • 区块菜单：右键区块可复制或删除整个区块。
  • 任务清单：使用 - [ ] / - [x] 创建可勾选的待办项。
  • 引用：使用 > 开头创建引用块。

表格

表格支持丰富的鼠标交互：

  • 编辑单元格：双击进入编辑。
  • 多选：按住 Ctrl（macOS 为 Command）点击多个单元格。
  • 合并 / 拆分：右键菜单对选中的单元格执行合并或拆分，以及增删行列。
  • 拖动重排：通过行列专用把手拖动，重新排列行或列。
已合并单元格可以和相邻单元格继续合并；若仅选到已合并区域的一部分，操作会被拒绝以保护表格结构。

代码块

代码块提供语言提示、展示态语法高亮并保留缩进：

  • 使用围栏语法 ```lang 指定语言，例如 ```go、```ts、```python。
  • 展示态下按语言进行语法高亮，编辑态回到纯源码。
  • 代码过长时，按住 Shift + 鼠标滚轮可水平浏览。

斜杠命令

在空区块输入 / 唤起命令菜单，快速插入常用结构：一级至三级标题、加粗、斜体、删除线、行内代码、链接、无序 / 有序 / 任务列表、引用、代码块与分割线。菜单支持中英文关键字过滤。

公式与图表

内建 KaTeX 数学公式与 Mermaid 图表渲染。行内与块级公式在展示态自动排版，Mermaid 代码块渲染为流程图、时序图等图形，适合技术文档。

主题与字号

提供三种主题：跟随系统、亮色、暗黑；正文字号可在 12–18px 之间调整。偏好通过本地存储持久化，桌面端与 Web 预览通用。

导出 HTML

可将当前文档导出为样式内联的独立 HTML 文件，无需外部资源即可在任意浏览器中打开，便于分享与归档。

AI 控制（MCP）

IdeaMarkdown 安装后即可被 AI 助手（通过 MCP）直接操作当前打开的 Markdown 文档——不用写一行代码，装完就用。

所有编辑都先进入 IdeaMarkdown 窗口中的内存文档模型，你能立刻在界面上看到变化，并可按 Ctrl + Z 撤销；AI 对文档的改动不会自动写入磁盘，只有调用 save_document 才会落盘。AI 生成的内容支持流式输出，逐字写入编辑器，无需等待完整响应。

工作原理与安全

每次启动 IdeaMarkdown 时，它会自动完成以下三步：

  1. 在随机端口上启动一个仅绑定到 127.0.0.1（本机）的 HTTP 控制服务。
  2. 生成一个全新的随机会话令牌。
  3. 将端口和令牌写入发现文件 %APPDATA%\com.ideamarkdown.app\control.json：
{ "port": 51234, "token": "<uuid>" }

MCP 服务器启动时会自动读取这个发现文件，然后向 http://127.0.0.1:<port>/control 发送请求，并带上 Authorization: Bearer <token>。整个过程对你完全透明——你不需要手动处理端口或令牌。

服务只接受本机回环连接，令牌每次启动都会轮换，旧令牌随应用重启立即失效。

不要把 control.json 里的令牌分享给不受信任的程序。
应用重启自动恢复：如果 IdeaMarkdown 重新启动了，ideamarkdown-mcp.exe 会在第一次请求失败后自动重读 control.json 并重试一次，通常不需要你手动干预。

MCP：让 AI 助手直接操作你的文档

MCP 服务器 ideamarkdown-mcp.exe 随安装包附带，位于安装目录。配置一次后，AI 助手就可以读取、修改、保存你当前在 IdeaMarkdown 中打开的文档。

一步一步来

总共只需 3 步，大约 2 分钟。

  1. 启动 IdeaMarkdown，并打开（或新建）一个 .md 文件。

  2. 配置 AI 客户端。打开你使用的 AI 工具的 MCP 配置文件（二选一），添加以下内容。注意把 <安装路径> 替换为 IdeaMarkdown 的实际安装目录：

    Cursor — 打开 Cursor 设置 → MCP → 找到 mcp.json：

    {
  "mcpServers": {
    "ideamarkdown": {
      "command": "<安装路径>\\ideamarkdown-mcp.exe"
    }
  }
}

    Claude Desktop — 打开 claude_desktop_config.json：

    {
  "mcpServers": {
    "ideamarkdown": {
      "command": "<安装路径>\\ideamarkdown-mcp.exe"
    }
  }
}
  3. 重启 AI 客户端，使其加载新配置。客户端启动后会看到 ideamarkdown 服务器处于已连接状态。

完成以上三步之后，你就可以在聊天中用自然语言指挥 AI 操作 IdeaMarkdown了。比如：

你（在 Cursor / Claude 的对话框中输入）：
"帮我在当前文档末尾插入一段 ## 待办事项 的标题，下面加三个任务"

或者：
"读取当前文档内容，帮我把里面的英文引号全部替换成中文引号"

AI 会自动调用对应的 MCP 工具来完成任务，IdeaMarkdown 窗口中会实时显示结果。

怎么确定安装路径？NSIS 安装程序默认安装到 C:\Users\<你的用户名>\AppData\Local\IdeaMarkdown\；如果你在安装时选择了其它位置，请以你实际选择的路径为准。不确定的话，可以在开始菜单搜索 IdeaMarkdown，右键「打开文件位置」即可看到安装目录。

MCP 可用工具

配置完成后，AI 助手可以使用以下 5 个工具来操作文档。你不需要手动调用它们——只要用自然语言告诉 AI 你想做什么，AI 会自动选择合适的工具。

工具参数作用
get_document 无 读取当前文档的完整 Markdown 内容、区块数量和当前活动区块索引。
insert_block markdown（必填）、position（可选）、after_block_index（可选） 在文档中插入一个或多个 Markdown 区块（标题、段落、列表等）。
insert_code_block lang（必填）、code（必填）、position、after_block_index 插入一段带语言标识的围栏代码块。适用于让 AI 帮你写代码示例。
replace_document markdown（必填） 用给定内容替换整个文档。适合让 AI 帮你重写或格式化整篇文章。
save_document 无 将当前文档保存到原路径。未保存过的新文档会弹出保存对话框。

position 参数说明

  • atEnd（默认值）：插到文档末尾。
  • atCursor：插到你当前光标所在位置。
  • afterBlock：插到指定区块之后；配合 after_block_index 使用（从零开始计数，省略时默认为 0）。

典型对话场景

# 让 AI 先看看你的文档写了什么
"帮我读一下 IdeaMarkdown 当前打开的文档，告诉我这段笔记有哪些可以改进的地方"

# 让 AI 在光标处插入内容
"在我光标位置插入一个## 注意事项的二级标题"

# 让 AI 帮你加一个代码示例
"在文档末尾插入一段 Python 代码，用快速排序的实现，叫它 quicksort，要有注释"

# 让 AI 重写整篇文档
"读取当前文档内容，把它改成英文版"

# 确认无误后落盘
"把 IdeaMarkdown 的当前文档保存到磁盘"

端到端示例：让 AI 在文档中插入 Go 快速排序

下面演示通过 MCP 让 AI 助手在打开的文档末尾插入一个 Go 语言快速排序代码块。

  1. 确保 IdeaMarkdown 正在运行，且已打开一个 .md 文档。
  2. 确保 MCP 服务器已按上文配置完成，AI 客户端（Cursor / Claude Desktop）已重启并加载了 ideamarkdown。
  3. 在 AI 对话框中输入："在 IdeaMarkdown 当前文档末尾插入一个 Go 语言的快速排序代码块"。
  4. AI 会调用 insert_code_block(lang="go", code="...", position="atEnd")。
  5. 代码块立即出现在 IdeaMarkdown 窗口中。如需撤销，按 Ctrl + Z。确认无误后让 AI 调用 save_document 落盘。

常见问题

为什么窗口只有部分区域能拖动？

应用采用无边框窗口，只有导航栏的空白背景区可以拖动窗口；菜单、文件名和窗口控制按钮保持可点击。

表格多选为什么用 Ctrl 而不是 Shift？

表格多选使用 Ctrl（macOS 为 Command），而不是 Shift。Shift + 滚轮用于代码块的水平浏览。

AI 客户端长时间连不上怎么办？

客户端会在首次失败后自动重读 control.json 并重试一次。若仍无法连接，请确认 IdeaMarkdown 正在运行，然后手动重启 MCP 服务器。
